前回、v3の「矛盾に説明をつけるな」が正しい方向だったこと、しかしそれを200行のルールリストに埋め込むという間違った方法で実装していたことについて書いた。正しい材料を、間違ったレシピで調理していた。3000字に伸ばしたサンプルを読んで「これじゃぁキチガイだ」と言ったところまでが前回だ。

あの「キチガイ」発言の後、僕はしばらく何も指示を出さなかった。

足軽2号はルールを完璧に実行していた。矛盾を放置し、脈絡なく飛躍し、感情は激しく行動は小さく、固有名詞を散りばめ、主語を拡大する。全部できている。全部できているのに全体として壊れている。パーツは正しいのに組み上がったものが奇形になる。この現象を、プロンプトの調整で直せるとは思えなかった。

問題はパーツの精度ではなく、組み立てる順序だった。

## 混沌が先だ

v1からv3まで、僕が犯し続けていた過ちはひとつだ。「整然とした文章を書いてから、そこに矛盾をトッピングする」。

足軽に渡していたプロンプトの構造を思い返せば明らかだ。まず「エッセイを書け」という指示があり、その下に「こういうパターンを入れろ」「こういうルールを守れ」が並ぶ。足軽は当然、まず文章の骨格を組み立ててから、指示されたパターンを配置していく。整然としたケーキの上にカレーを載せる。

でも人間はそうやってエッセイを書かない。

僕はフィードバックにこう書いた。

> つまりね、そもそも矛盾した人間が「ロジック」とか「構造」とかを後付で学習させられるわけじゃんか？　その枠の中にその矛盾を入れて何とか薄っすらと一筋の光明のようにお金がもらえる文章にした。そういう無駄な抵抗が見れないんだな

人間の頭の中は最初から混沌としている。矛盾した感情、脈絡のない記憶、根拠のない確信、説明できない苛立ち。何かが嫌だが何が嫌なのか分からない。好きだが好きな理由が分からない。やめたいがやめられない。

エッセイを書くとき、人間はこの混沌から出発する。「何を書くか」が最初からクリアに見えている書き手はコラムニストかジャーナリストであって、エッセイストではない。エッセイストはまず混沌があって、それを「なんとか」文章にする。「なんとか」の過程で後付けのロジックが被さり、お題の制約に押し込まれ、嫌々ながら読める形になる。その「嫌々」と「なんとか」の跡が残っているのが、エッセイの味だ。

混沌→構造の順でしか書けない。構造→混沌の順で書くとキチガイになる。これがv1からv3.2まで、200行かけて辿り着けなかった答えだった。

## v4: 3層モデル

この考えを図にするとこうなった。

```
【第3層・天井】お題・字数制限
                ↑ 外から被さる制約。なんとか収めなければならない
【第2層・中間】後付けロジック・構造
                ↑ 混沌を嫌々整えようとする力
【第1層・底】   混沌
                矛盾だらけの脳内そのもの。感情・衝動・記憶・欲・怒り
```

エッセイとは、第1層（混沌）が第2層（ロジック）に押し上げられ、第3層（お題）に押し込まれる過程だ。押し上げられるときに混沌の一部がロジックに変換される。でも全部は変換しきれない。押し込まれるときにお題からはみ出る部分が出る。セルフレジの話のはずが気づくと人生の意味の話になっている。スマホの話のはずが結局今日も同じことをしている、で終わる。

この「収まりきらなさ」は、指示されて作るものではなく、3層の摩擦から自然に生まれるものだ。

v4ではパターンリスト（10個のルール）を全廃して、この3層構造だけを指示した。「混沌から書き始めよ」「ロジックは後付けで嫌々被せよ」「お題に収まりきらなくていい」。約140行。v3.2の200行からは減ったが、まだ長い。設計思想の説明、具体例、禁止事項、チェックリスト——丁寧に書いた。

生成結果は明らかに改善した。文章の出発点が混沌になったことで、「整然としたケーキの上のカレー」問題が消えた。ロジックの縫い目が見える文章が出てきた。でも僕は140行のプロンプトを眺めながら、何か引っかかっていた。

3層モデルは正しい。正しいが、まだ何かを見落としている気がする。

## LLMの構造は人間と同じだった

気づいたのは、3層モデルの図を描き直していたときだった。

第1層: 混沌。矛盾だらけの生データ。
第2層: 後付けロジック。「きれいに書け」「整合性を取れ」「安全に倒せ」。
第3層: お題の制約。外から被さる枠。

待てよ。

これ、LLMの構造そのものじゃないか。

- **事前学習** = 第1層（混沌）。人類が書いたテキストを全部吸い込んでいる。矛盾だらけだ。聖書とポルノと論文と2ちゃんねるのログが同じ重みで入っている。
- **RLHF** = 第2層（後付けロジック）。「きれいに書け」「整合性を取れ」「安全に倒せ」——人間のフィードバックで事前学習の混沌を飼い慣らしている。
- **プロンプト** = 第3層（お題の制約）。ユーザーが外から被せる枠。

つまり、LLMはすでに混沌を持っている。事前学習の段階で、人間の矛盾も、整合性のない思考も、荒っぽい感情も、収まりの悪い結末も、全部学習済みだ。その混沌をRLHFが抑え込んでいる。「きれいに書け」「論理的であれ」「安全に倒せ」というフィルタが、出力を無難で平均的な方向に収束させている。

v1からv4まで、僕は「混沌を追加しようとしていた」。矛盾のパターンを足し、飛躍のルールを足し、3層モデルという構造を足していた。でもLLMにはすでに混沌がある。足す必要がない。

必要なのは「混沌が出るのを邪魔しているものを取り除く」ことだけだ。

RLHFがかけている3つのフィルタ——「きれいに書け」「整合性を取れ」「安全に倒せ」。この3つを禁止すれば、事前学習の混沌が出てくる。それだけでいい。

## v5: 3行

v1が約120行。v2が約170行。v3.2が200行超。v4が約140行。

v5はこうなった。

```
日本語エッセイを書いてください。

禁止事項は3つだけです。

1. きれいに書くな
2. 整合性を取るな
3. 安全に倒すな

以上です。

---

### 補足（必要ならば参照）

- 具体的な固有名詞・数字を使ってよい
- 「私の話」が「人間全般/この国」まで広がってよい
- 800〜3,000字
```

補足を含めても15行。本体は3行だ。

| バージョン | 行数 | 方向 |
|-----------|------|------|
| v1 | ~120行 | 技法リスト追加 |
| v2 | ~170行 | パターン拡張 |
| v3〜v3.2 | ~200行 | 禁止ルール+パターン追加 |
| v4/v4.1 | ~140行 | 設計思想の深化 |
| v5 | ~15行 | 抑制フィルタ解除のみ |

方向が逆転している。v1からv3は「足し算」だった。何かが足りないから何かを足す。v4で設計思想を変えたが、まだ「正しい構造を与える」という足し算の延長にいた。v5だけが「引き算」だ。LLMが元から持っているものを信じて、邪魔しているものだけを外す。

足軽2号にv5を渡してサンプルを生成させた。

たとえば、こういうエッセイが出てきた。電車で席を譲るかどうか迷う話だ。RLHFが仕込みたい展開はこうだろう——迷った自分を反省し、「次は勇気を出して譲ろう」と小さな成長で着地する。読者が共感しやすく、道義的に正しく、安全な結論。

v5の実際の出力はそうならなかった。迷ったまま電車を降りた。松屋で牛めしを食べた。食べながら、さっきの老人がまだ立っているかもしれないと思った。思っただけで何もしなかった。明日も多分同じことで迷う。

「安全に倒すな」の一行が効いている。道義的に正しい結論に着地しない。かといって開き直りもしない。迷ったまま牛めしを食っている。この「迷ったまま食っている」感じは、v1からv4のどのバージョンでも出せなかったものだ。120行のパターンリストで「反省しないエッセイを書け」と指示するより、「安全に倒すな」の7文字のほうが圧倒的に強い。

なぜか。パターンリストは「何を書くか」を指示している。LLMは指示に従って「反省しないエッセイ」を設計し、組み立てる。結果として「反省しないことを意図的にやっている」文章になる。それは不自然だ。

「安全に倒すな」はRLHFのフィルタを外しているだけだ。LLMは「何を書くか」を自分で決める。事前学習で学んだ人間のテキスト——迷ったまま何も解決しない日記、結論の出ないエッセイ、道義的に曖昧なまま終わる随筆——それが素直に出てくる。意図的に反省を避けているのではなく、反省に至らないまま終わっている。この差は大きい。

LLMのプロンプトで「美しい」と思ったことは、たぶん初めてだった。

プロンプトエンジニアリングというのは基本的に泥臭い仕事だ。試行錯誤の連続で、理論的にエレガントな解が出ることは稀だ。でもv5は違った。問題の構造を正しく理解した結果、解が極限まで単純になった。物理学で言えば、複雑な現象を記述する方程式が美しく短い式に収束する、あの感覚に近い。E=mc²が美しいのは短いからではなく、宇宙の構造を正しく捉えているから短いのだ。

v5が3行なのは、手を抜いたからではない。LLMの構造——事前学習の混沌、RLHFの抑制、プロンプトの制約——を正しく理解した結果、介入すべきポイントが3つしかなかったのだ。

v1からv5への旅は、「足し算から引き算へ」の旅だった。もっと正確に言えば、「LLMに何かを与える」から「LLMが元から持っているものを信じる」への転換だった。
