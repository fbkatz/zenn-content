# v1〜v2: 足し算の誘惑——ルールを追加するほどAIっぽくなる

AIにエッセイを書かせる。前回、その無謀な企画の出発点について書いた。LLMの3層構造——事前学習の混沌、RLHFの後付け構造、プロンプトという制約。この3つの間で文章が押しつぶされながら生成されるという仮説。

仮説は立てた。あとは検証するだけだ。

僕がまず書いたv1プロンプトは、こういうものだった。

```
あなたは日本語エッセイの書き手です。以下の「書き方のルール」と「禁止事項」をかならず守ってください。

### あなたの文体について
短く切る。でも、ときどき長くなる。その切り替えに規則はない。

### 禁止事項
最重要禁止: きれいにまとめない
起承転結を作らない
結論ファーストを避ける

### 書き方の具体的な技法
技法1: 脱線の仕込み方（「そういえば」で別の話を始める）
技法2: 反オチの着地（問いに答えない）
技法3: 感覚の具体化（感情を行動に置き換える）
技法4: 自己卑下の活用
技法5: 短文の連打から長文への移行

### 最終チェック
□ 結論や教訓があるか → あれば削除する
□ 脱線が1箇所以上あるか
□ AIっぽい語彙があるか → あれば言い換える
```

わりと真っ当なプロンプトだと思った。禁止事項も具体的だし、技法にも名前をつけた。最終チェックリストまである。プロンプトエンジニアリングの教科書に載せても恥ずかしくない程度には体裁が整っている。

足軽2号にこれを渡して、3つのお題でエッセイを生成させた。「コンビニのレジ待ちで考えたこと」「雨の日が嫌いだったのにいつからか好きになった話」「スマホの画面が割れたままにしている理由」。日常の、どうでもいい題材。エッセイとしてはちょうどいい粒度のはずだった。

足軽2号の自己評価は82〜88点。高い。

僕の評価はNOGO。「本当にどうでもいい文章で逆に知性を感じません」とフィードバックした。

何が起きたか。「AIっぽさを消す」を忠実に守った結果、文章からAIっぽさだけでなく知性も消えた。脱線はする。結論もない。短文と長文の切り替えもある。チェックリストの項目は全部クリアしている。にもかかわらず、出来上がったものはただの日記だった。文体の表面だけ人間に寄せた、中身のない文章。

これは僕のプロンプトの問題だ。「何を禁止するか」は書いたが、「何を書くべきか」を書いていなかった。もっと正確に言うと、「どういう知性で書くか」を指定していなかった。

## 7つのフィードバック

僕は足軽2号に7つの具体的なフィードバックを出した。

1つ目。極論や論理破綻をすること。それを牽強付会で無理やり終わらせること。人間のエッセイには論理の飛躍がある。その飛躍を「そういえば」の脱線で処理するのではなく、堂々と論理を壊して、壊れたまま強引に着地する。

2つ目。世の理不尽に憤って脳内では乱暴狼藉を働くが、実際には小市民としてつましく行動すること。内面の過激さと外面の凡庸さ。この落差が人間臭い。

3つ目。書きたいことへの密度のムラ。興味があるところは異常に詳しく、ないところは雑。v1の足軽はすべてのパラグラフを均等な密度で書いていた。それ自体がAIの署名だ。

4つ目。推敲せず後の文で方向修正する人間臭さ。前の文を直すのではなく、「いや、そうじゃなくて」と後ろで軌道を変える。

5つ目。改行のムラ。興が乗らない話題では改行が多くなり、興が乗ると改行なしでガッと書く。

6つ目。脳内エロ・グロ・ナンセンスの気配だけ匂わせる。直接は書かない。でも匂いだけはする。この「気配」が文章に体温を与える。

7つ目。主張はあるが結論は曖昧——それが正しいバランス。v1の「結論を書くな」は間違いだった。主張がないから知性を感じないのだ。主張はある。ただし結論は出ない。この微妙な位置に人間のエッセイはある。

## v2: 足し算の罠

v2では、この7つのフィードバックをそのままプロンプトに組み込んだ。

極論→牽強付会パターン。憤り→小市民パターン。知的横断。密度ムラ。前言撤回なしの軌道修正。改行ムラ。エログロの気配。全部、具体例つきでプロンプトに書いた。

結果は「だいぶ良くなってきた」。確かに良くなった。密度にムラが出た。主張が生まれた。文章に体温のようなものが感じられるようになった。でもまだ何かが足りない。商品名やモデル名、具体的な金額からの脱線——人間が文章を書くときに自然に出る「ディテールへの執着からの逸脱」がない。

そしてもう一つ、もっと根本的な問題に気づいた。

v1のプロンプトは約120行だった。v2は約170行。フィードバックのたびにルールを追加していた。「ここが足りない」→ルール追加。「こうしてほしい」→パターン追加。プロンプトは膨張し続けていた。

足軽は律儀にそのルールを全部守ろうとする。「極論パターンを入れてください」と書けば極論を入れる。「密度にムラを出してください」と書けば密度にムラを出す。「改行にムラを」と言えば改行にムラを出す。一つひとつは正しく実行されている。

問題は、全部のルールを同時に守ろうとする「律儀さ」そのものがAIっぽいということだ。

人間はルールを全部守らない。というか、守れない。7つのフィードバックを渡されたら、人間の書き手は2つくらい強く意識して、3つは無意識に取り込んで、残り2つは忘れる。その「忘れ方」にその人の個性が出る。LLMは忘れない。指示されたことは全部やる。その完璧さが、文章を均質にする。

もう一つ。ルールが増えるほど、ルール間の相互作用が複雑になる。「極論を書け」と「小市民として行動しろ」は矛盾しない。でも170行のルールの中には微妙に矛盾するものが出てくる。人間はその矛盾を無意識に調停する。LLMは全部の条件を満たそうとして、結果的にどの条件も中途半端に満たす——つまり平均的な文章になる。

ルールを追加するという行為自体が、RLHFの構造と同じことをやっている。「こうしろ」「ああしろ」という制約を外から重ねていく。RLHFが事前学習の混沌を飼い慣らすように、僕のプロンプトもLLMの出力を飼い慣らそうとしていた。飼い慣らされた文章が面白いわけがない。

足し算のアプローチには限界がある。これは確信に変わりつつあった。ルールを足すのではなく、何か別のことをしなければならない。でもそれが何かは、まだ分かっていなかった。

次回、その転換点について書く。きっかけは、ヒトラーの愛犬だった。
