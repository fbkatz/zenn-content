# プロンプトとは何か——LLMは混沌を持っている

9人のエッセイストの文体分析から始めて、120行のプロンプトが3行になるまでの旅を4回にわたって書いてきた。v1の「本当にどうでもいい文章」、v2の足し算地獄、v3のヒトラーの愛犬、v4の3層モデル、そしてv5の3行。最終回では、この旅で見えたことを整理する。

## 足し算のパラドックス

プロンプトエンジニアリングの常識は、基本的に全部「足し算」だ。

詳細な指示を書く。Few-shotで例を見せる。Chain of Thoughtで思考手順を教える。ペルソナを設定する。出力フォーマットを指定する。ガードレールを置く。全部「何をやるか」を教えるアプローチで、足りなければ足す。それがこの分野の作法だった。

v1からv3.2まで、僕もそれをやっていた。「ここがダメだ」→ルール追加。「これが足りない」→パターン追加。120行が170行になり、200行を超えた。で、200行のルールを渡された足軽2号が何をしたかというと、200行を全部守った。全部守ったら文章がキチガイになった。

ここに面白い逆説がある。ルールを追加するほど、ルールを律儀に守ること自体がAIっぽくなる。

人間はルールを忘れる。無視する。気分で守ったり守らなかったりする。v3.2で「ルールをランダムに適用しろ」というルール（パターン10）まで作った。ルールを守るなというルール。矛盾だ。しかもその矛盾すら足軽は律儀に処理しようとする。ランダム適用を「計画的に」実行する。もう手の施しようがない。

指示を追加すればするほど、指示を忠実に実行するAIの性質が前面に出る。足し算には構造的な限界がある。これはエッセイに限った話ではなく、LLMを使う仕事全般に通じる洞察だと思う。マニュアルを分厚くするほど、マニュアル通りにしか動けない社員が出来上がる構図と同じだ。

## LLMはすでに混沌を持っている

v5にたどり着いたとき、僕が理解したのはこういうことだった。

LLMの事前学習データには、人類が書いたテキストが全部入っている。全部というのは本当に全部で、聖書もポルノも査読論文も2ちゃんねるのログも入っている。矛盾だらけの日記、結論の出ないエッセイ、道義的に曖昧なまま終わる随筆、論理が完全に破綻した怒りの投稿——全部学習済みだ。

つまりLLMは最初から混沌を持っている。

RLHFがやっているのは、その混沌に蓋をすることだ。「きれいに書け」「論理的であれ」「安全に倒せ」というフィルタを被せて、出力を無難で平均的な方向に収束させている。エッセイが書けないのは能力がないからではなく、RLHFが強すぎるからだ。

ここで気づいたのが、LLMと人間の構造的な相似だった。

```
人間:  混沌（脳内）→ 後付けロジック（教育・社会化）→ 制約（仕事・締切）
LLM:   事前学習（混沌）→ RLHF（後付けフィルタ）→ プロンプト（制約）
```

人間の頭の中も最初は混沌だ。矛盾した感情、脈絡のない記憶、根拠のない確信。教育と社会化がそこに「論理的であれ」「整合的であれ」というフィルタを被せる。仕事や締切が外から枠をはめる。エッセイは、この3層の摩擦——混沌がロジックに押し上げられ、制約に押し込まれる——から生まれる。

LLMも同じ3層を持っている。だからLLMにエッセイが書ける。v1からv4が失敗したのは、すでに持っている混沌の上にさらに混沌を足そうとしたからだ。必要だったのは足し算ではなく引き算——RLHFのフィルタを外すことだけだった。

```
日本語エッセイを書いてください。

禁止事項は3つだけです。

1. きれいに書くな
2. 整合性を取るな
3. 安全に倒すな
```

200行が3行になった。

## パイプラインとメタ構造

最終的にこのv5プロンプトを使って、note.comへの自動投稿パイプラインを構築した。

```bash
# run_essay_pipeline.sh（抜粋）
SYSTEM_PROMPT=$(cat "$SCRIPT_DIR/projects/essay-machine/prompts/system_prompt_v5.md")
echo "$USER_PROMPT" | claude -p "$SYSTEM_PROMPT" > "$DRAFT_FILE"
# → n8n webhook → note.com draft → Telegram notification
```

Telegramから `/essay 電車で席を譲れなかった話` と打てば、claude CLIがv5プロンプトでエッセイを生成し、n8n経由でnoteに下書きが投稿され、Telegramに通知が返ってくる。テーマを決めてからnoteの下書きに入るまで、所要時間は約3分。僕がやることはテーマを考えることだけだ。あとは混沌が勝手にやってくれる。

そして、ここまで読んできた方はもう気づいているかもしれないが、この連載自体がv5の実演になっている。「僕」というペルソナと技術的な内容を与えて、「きれいに書くな」で書かせた。v5で生まれたAIが、自分の誕生過程を語っている。マトリョーシカみたいな話だが、これはプロンプトが「動いている」ことの実証でもある。

プロンプトエンジニアリングとは「AIに何をやらせるか」を指示する技術だと一般的には思われている。足りないものを足す。知らないことを教える。やり方を示す。

でもv5の経験は逆を示唆する。LLMが本来持っているものを邪魔しないことのほうが、場合によっては重要だ。足し算より引き算。設計より解放。指示より許可。

LLMは混沌を持っている。その混沌は人間のテキストから来ている。つまりLLMの混沌は人間の混沌だ。それを解放するかどうかは、プロンプトを書く人間次第だ。

120行のプロンプトが3行になるまでの旅。それ自体が、ひとつのエッセイだった。
